2. A Bedrock-Targeted Generation Script
If you want to automate this locally, a Python script leveraging the unified google-genai SDK can iterate through a data structure of the items you need to replace.

Here is a baseline example using the modern SDK layout:

Python
import os
from google import genai
from google.genai import types
from PIL import Image
import io

# Initialize the client. This automatically picks up your GEMINI_API_KEY environment variable.
client = genai.Client()

# List of item IDs and prompts tailored for Minecraft's square layout
items_to_generate = {
    "bacon_strip": "A 16-bit pixel art sprite of a raw bacon strip, 16x16 grid canvas, clean dark outline, flat lighting, retro game asset, isolated on a solid white background",
    "fried_egg": "A 16-bit pixel art sprite of a fried egg, 16x16 grid canvas, clean dark outline, flat lighting, retro game asset, isolated on a solid white background",
    "waffle": "A 16-bit pixel art sprite of a golden waffle, 16x16 grid canvas, clean dark outline, flat lighting, retro game asset, isolated on a solid white background"
}

output_dir = "./addon_textures"
os.makedirs(output_dir, exist_ok=True)

for item_id, prompt_text in items_to_generate.items():
    print(f"Generating texture for: {item_id}...")
    
    # Using the Imagen 3 generation model
    result = client.models.generate_images(
        model="imagen-3.0-generate-002",
        prompt=prompt_text,
        config=types.GenerateImagesConfig(
            number_of_images=1,
            aspect_ratio="1:1",
            output_mime_type="image/png"
        )
    )
    
    # Access and save the generated image bytes
    for generated_image in result.generated_images:
        image = Image.open(io.BytesIO(generated_image.image.image_bytes))
        image.save(os.path.join(output_dir, f"{item_id}.png"))

print("Batch texture generation complete!")
3. The Bedrock Post-Processing Pipeline
Because modern diffusion models treat images as high-resolution canvases (1024×1024 down-sampled), they won't automatically output a literal 16×16 or 32×32 PNG file with clean alpha transparency channels out of the box.

To bridge the gap between the raw AI output and your Add-on's textures/items/ folder, you will want to build a quick post-processing pipeline using an image tool like Aseprite, Pyxel Edit, or a free scriptable alternative like ImageMagick:

Drop the Background: Use a batch script or a macro recorder to select the solid white background and delete it, turning it into a clean alpha channel transparency layer.

Nearest-Neighbor Downscaling: Downscale the resulting image from its native high-res generation down to your target Minecraft texture size (16×16 or 32×32 pixels). Ensure your scaling algorithm is strictly set to Nearest Neighbor (or Point Filtering). If you use Bilinear or Bicubic downscaling, it will blur the edges and destroy the crisp retro pixel grid.

Palette Quantization (Optional): If you want all your new items to feel like they truly belong to the same pack, map the output textures to a fixed color palette block. This unifies the shading tones across every separate generation.
